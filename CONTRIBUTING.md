# Contributing to Open-AI-Moving-Checklist

Thanks for your interest in contributing! This project is all about helping people take control of their data.

## How to Contribute

### Reporting Issues

Found a bug or have a suggestion?
1. Check if the issue already exists
2. Open a new issue with a clear description
3. Include steps to reproduce (for bugs)
4. Share your OpenAI export format (if relevant, without sensitive data)

### Submitting Changes

1. Fork the repository
2. Create a new branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test your changes with example data
5. Commit with clear messages
6. Push to your fork
7. Open a Pull Request

### Code Style

- Follow PEP 8 for Python code
- Add comments for complex logic
- Keep functions focused and small
- Update documentation for new features

### Testing

Before submitting:
- Test with the example data
- Verify output format is correct
- Ensure no sensitive data is logged
- Check that errors are handled gracefully

### Documentation

- Update README.md if adding features
- Add examples for new functionality
- Keep docs clear and concise

## What We're Looking For

- **New export formats**: Support for different OpenAI export structures
- **Better extraction**: Improved algorithms for finding persona data
- **Security**: Encryption, secure storage options
- **Migration tools**: Scripts to move data to specific platforms
- **Bug fixes**: Always welcome!

## Code of Conduct

- Be respectful and constructive
- Focus on the code, not the person
- Help others learn and grow
- Keep discussions on-topic
